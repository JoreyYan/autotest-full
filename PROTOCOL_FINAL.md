## 二、波特率选择（重要）

### 9600 bps vs 115200 bps

| | 9600 bps | 115200 bps |
|--|---------|-----------|
| 配置加载耗时 | ~12秒 | **~2秒** |
| 单次测试耗时 | ~2.8秒 | **~1.7秒** |
| 数据格式 | CSV 文本 | **CSV 文本** |
| 稳定性 | 有坑（见下） | **稳定** |

### 9600 bps 的坑（已验证）

```
mmem:load:trs 2 → 等15秒（等全部传完）→ 发 *TRG → ❌ 无响应
mmem:load:trs 2 → 等 2秒（传输中）   → 发 *TRG → ✅ 正常
```

等配置文件完整传输结束后再发 `*TRG`，仪器不响应。原因不明，属于 9600 bps 特有问题。

### 结论：使用 115200 bps

115200 bps 完全没有这个问题，而且更快，推荐使用。

## 三、核心通讯流程

### 3.1 正确流程（已验证）

```
程序启动时（只做一次）：
  电脑 → 仪器:  mmem:load:trs 2\r\n
  仪器 → 电脑:  [11264字节 配置文件，约2秒传完，直接丢弃]

之后每次测试：
  电脑 → 仪器:  *TRG\r\n
  仪器 → 电脑:  [第1包] !  (1字节，约0.4~0.7秒后到)
  仪器 → 电脑:  [第2包] CSV数据  (约1554字节，约1.7秒后到)
```

### 3.2 错误流程（不要用）

```
❌ 方式1：*TRG → *OPC? 轮询 → TRS:DATA? 查询
   → 这是后来写的代码，是错的。TRS:DATA? 在115200返回二进制，无法解析。

❌ 方式2：mmem:load:trs 2 → 等15秒 → *TRG（仅9600有此问题）
   → 9600 bps下，等配置文件全传完再触发，仪器不响应。
```

## 四、关于 mmem:load:trs 2 命令

### 这条命令做了什么

1. **告诉仪器切换到2号配置**（若已在2号，就是确认）
2. **把2号配置文件内容发回电脑**（11264字节，我们不需要，直接丢弃）

### 重要认知

- 配置**存储在仪器内部**，不需要电脑写入
- `mmem:load:trs 2` 只是切换/确认，以及返回配置数据（供电脑读取，我们用不到）
- 如果只测一种产品，理论上可以不发这条命令，仪器保持上次的配置

### 什么时候需要发

- 程序首次启动时，确保仪器用的是正确配置
- 需要切换产品型号（比如从2号切换到3号配置）时

### 切换配置的时序要求

```python
# ✅ 正确：发完等配置包收完就行
collector.send('mmem:load:trs 2', ending='\r\n')
collector.recv_packet(timeout=5.0)  # 等配置包收完，丢弃
# 之后随时可以发 *TRG，不限时间
```

## 六、返回数据格式

### 6.1 第1包：确认字节

```
!
```
- 1个字节，ASCII 感叹号（0x21）
- 含义：仪器收到 `*TRG`，开始执行扫描测试

### 6.2 第2包：CSV 测试数据

**约1554字节，纯文本，可打印率99.9%**

格式：
```
!1,Turn,'3-1,+1.000000e+00,0.000000e+00,0.000000e+00,Pass;
1,Turn,'4-5,+1.453248e-01,1.000000e-01,2.000000e-01,Pass;
1,Lx,'3-1,1.032975e-03,9.000000e-04,2.000000e-03,Pass;
...（共28条）
```

- 每条记录用 **分号 `;`** 分隔
- 第一条记录前有 `!`（解析时去掉）
- 字段用 **逗号 `,`** 分隔

### 6.3 字段说明

```
字段顺序：序号, 测试类型, 引脚, 测量值, 下限, 上限, 结果

示例：1, Lx, '3-1, 1.032975e-03, 9.000000e-04, 2.000000e-03, Pass
```

| 字段 | 说明 | 注意 |
|------|------|------|
| 序号 | 固定为1 | |
| 测试类型 | Turn/Lx/Q/Lk/Cx/Dcr | |
| 引脚 | `'3-1` 前有单引号 | 解析时用 `.lstrip("'")` 去掉 |
| 测量值 | SI单位，科学计数法 | 见单位换算表 |
| 下限 | 0表示不限制 | |
| 上限 | 0表示不限制 | |
| 结果 | `Pass` 或 `Fail` | 仪器自身判定 |

### 6.4 测量值单位换算

| 测试类型 | 仪器返回单位 | 显示建议 |
|---------|------------|---------|
| Turn | 无量纲（比值） | 直接显示 |
| Lx | H（亨利） | × 1000 → mH |
| Q | 无量纲 | 直接显示 |
| Lk | H（亨利） | × 1,000,000 → μH |
| Cx | F（法拉） | × 1e12 → pF |
| Dcr | Ω（欧姆） | 直接显示 |

### 6.5 怎么知道传输完了

SerialCollector 使用 **300ms空闲封包**机制：

```
仪器持续发送数据块（~60字节/50ms，115200baud）
  ...最后一块发完...
  ← 300ms 无新数据 → SerialCollector 封包 → 放入队列
```

- 1554字节 ÷ 11520字节/秒 ≈ 0.13秒传输
- 加300ms空闲判定 → 约0.4秒后封包完成
- 可靠：仪器发完自然停，300ms足够判定

## 七、完整代码实现

### 7.1 SerialCollector 配置

```python
from serial_collector import SerialCollector

collector = SerialCollector(
    port='COM4',
    baudrate=115200,
    idle_ms=300      # 300ms空闲 = 一包结束
)
```

### 7.2 初始化（程序启动时执行一次）

```python
collector.start()
time.sleep(0.3)
collector.clear_queue()

# 加载仪器配置（等配置包收完即可，不需要sleep）
collector.send('mmem:load:trs 2', ending='\r\n')
cfg_data = collector.recv_packet(timeout=5.0)  # 等收完，丢弃
# 此后随时可以测试
```

### 7.3 每次测试

```python
# 触发
collector.send('*TRG', ending='\r\n')

# 第1包：确认
ack = collector.recv_packet(timeout=3.0)   # b'!'

# 第2包：CSV数据
raw = collector.recv_packet(timeout=10.0)  # ~1554字节

# 解析
text = raw.decode('ascii', errors='replace')
records = []
for seg in text.split(';'):
    seg = seg.strip().lstrip('!')
    parts = seg.split(',')
    if len(parts) >= 7:
        try:
            records.append({
                'type':   parts[1],
                'pins':   parts[2].lstrip("'"),
                'value':  float(parts[3]),
                'lo':     float(parts[4]),
                'hi':     float(parts[5]),
                'result': parts[6].strip()   # 'Pass' or 'Fail'
            })
        except:
            pass
```

### 7.4 收尾

```python
collector.close()
```

## 十、踩过的坑（避免再踩）

### 坑1：*OPC? + TRS:DATA? 方式不对

```python
# ❌ 错误：这是历史错误代码
*TRG → wait_opc() → TRS:DATA? → 收数据
```

这种方式下：
- `TRS:DATA?` 在115200返回**二进制**（`#\x01<float数组>`）
- `TRS:DATA?` 在9600返回二进制混杂OPC残留
- 需要按偏移量解析float，麻烦且不稳定

### 坑2：9600 bps 等15秒后 *TRG 失效

原因不明，疑似仪器等配置全传完后进入某个状态。
**解决：用115200，问题不存在。**

### 坑3：CSV第一条前有 `!`

```
原始数据：!1,Turn,'3-1,...;1,Turn,'4-5,...
         ↑ 注意这里有!
```

解析每条记录时需要 `.lstrip('!')` 去掉。

### 坑4：引脚字段前有单引号

```
'3-1  →  解析时用 .lstrip("'") 去掉
```

### 坑5：波特率改变影响了历史测试结果

历史上有一段时间用9600，有一段用115200，导致部分旧代码和文档描述不一致（有的说CSV，有的说二进制）。
**现在统一用115200，统一用 *TRG 两包方式，返回CSV。**

## 十一、关于仪器内部配置

- 配置**存在仪器内部**，最多100组（1号~100号）
- 每组配置包含：脚位设置、测试项开关、频率、电压、上下限等
- **不能从电脑写入配置**（已验证 `mmem:store:trs` 无响应）
- 配置只能在**仪器屏幕**上手动设置
- `mmem:load:trs N` 只是读取+切换，不是写入
