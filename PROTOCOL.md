# UC2866XB / UC2869 通讯协议说明

## 硬件连接

| 参数 | 值 |
|------|-----|
| 串口号 | COM4（USB转串口，PL2303GT） |
| 数据位 | 8 |
| 停止位 | 1 |
| 校验位 | 无 |
| 流控 | 无 |

---

## 波特率与数据格式的关系（关键！）

| 波特率 | TRS:DATA? 返回格式 | 加载配置耗时 | 每次测试耗时 |
|--------|-------------------|------------|------------|
| **9600** | **CSV 文本**（推荐） | ~12 秒 | ~4 秒 |
| 115200 | 二进制 float 数组 | ~2 秒 | ~2 秒 |

> **结论：使用 9600 bps，TRS:DATA? 返回可直接解析的 CSV 文本。**
> 115200 bps 下返回二进制，需要按偏移量解析 float 数组。

---

## 指令序列（9600 bps，推荐方案）

### 初始化（只做一次）

```
1. 连接串口 COM4, 9600, 8N1
2. 发送: mmem:load:trs 2\r\n
3. 等待 2 秒
4. 清空接收队列（仪器返回 ~11264 字节二进制配置文件，丢弃）
```

### 每次测试

```
1. 发送: *TRG\r\n
   → 仪器立即回: ! (0x21，表示"开始测试")
2. 轮询: *OPC?\r\n，每 300ms 一次
   → 仪器回: 1\n 表示测试完成（约 1.5 秒）
3. 清空队列（清除 OPC 轮询产生的残留 1\n）
4. 发送: TRS:DATA?\r\n
   → 仪器返回 CSV 文本（约 3853 字节，29 条记录）
5. 解析 CSV
```

---

## TRS:DATA? 返回的 CSV 格式

### 格式结构

每条记录用 **分号 `;`** 分隔，字段用 **逗号 `,`** 分隔：

```
seq,test_type,'pins,measured,lower_limit,upper_limit,result,timestamp;
```

### 字段说明

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| seq | int | 序号 | 1 |
| test_type | str | 测试类型 | Turn / Lx / Q / Lk / Cx / Dcr |
| pins | str | 引脚对（前有单引号） | '3-1 |
| measured | float | 测量值（SI单位） | +1.000000e+00 |
| lower_limit | float | 下限（0 = 不限制） | 1.000000e-01 |
| upper_limit | float | 上限（0 = 不限制） | 2.000000e-01 |
| result | str | 仪器判定结果 | Pass / Fail |
| timestamp | str | 测试时间 | 2025-12-22 14:18:54 |

### 实际数据示例（ZZ-T250005A，28条）

```

1,Turn,'6-7,+1.664335e-01,1.000000e-01,2.000000e-01,Pass,2025-12-22 14:18:54;
```

### 测量值单位（SI 原始值）

| 测试类型 | 单位 | 换算显示 |
|---------|------|---------|
| Turn | 无量纲（比值） | 直接显示 |
| Lx | H（亨利） | ×1000 → mH |
| Q | 无量纲 | 直接显示 |
| Lk | H（亨利） | ×1e6 → μH |
| Cx | F（法拉） | ×1e12 → pF |
| Dcr | Ω（欧姆） | 直接显示 |

## 串口接收参数

| 参数 | 值 | 说明 |
|------|-----|------|
| serial timeout | 0.05s | 每次 read() 等待时间 |
| idle_ms | **300ms** | 无新数据多久视为一包结束 |
| recv timeout | 5.0s | 等待数据包的最长时间 |

> **idle_ms 必须是 300ms**，不能太长（会导致 OPC 轮询无法封包）也不能太短（会切断数据）。

## 仪器内部配置文件

### 加载命令

```
mmem:load:trs N    (N = 1~100)
```

- 仪器返回该配置的二进制文件（~11264 字节），**直接丢弃**，不需要解析
- 同时仪器切换到该配置，下次 *TRG 使用此配置测试

### ZZ-T250005A 对应关系

| 产品 | 仪器配置文件编号 |
|------|---------------|
| ZZ-T250005A | **2** |

### 配置文件内容（已确认）

- 初级：1 组
- 次级：5 组
- 引脚总数：14
- 脚位：3-1, 4-5, 6-7, 8-10, 13-12, 14-13
- 启用测试：Turn / Lx / DCR / Lk / Cx

---

## 注意事项

1. **命令结束符必须是 `\r\n`（CRLF）**，不能只用 `\n`
2. **命令间至少间隔 50ms**，发太快仪器会乱
3. **`mmem:load:trs` 只需初始化时加载一次**，之后连续测试不需要重新加载
4. **`TRS:DATA?` 返回数据末尾可能带有二进制乱码**，解析时遇到非打印字符截断即可
5. **仪器不支持从电脑上传配置**（已验证 mmem:store:trs 无响应）

## 调试确认（2026-03-07）

- 端口：COM4（已实测）
- 115200 → 二进制格式（已实测）
- 9600 → CSV 格式（有历史捕获文件 query_response_correct.bin 为证）
- `*TRG` 立即返回 `!`（1字节，确认收到）
- `*OPC?` 约 1.5 秒后返回 `1`（测试完成）
