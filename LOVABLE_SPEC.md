# 变压器自动测试系统 — Lovable 前端设计规格

## 项目背景

工厂现场使用的变压器自动测试系统。操作员站在工作台旁，把变压器放上夹具，点一次按钮，系统自动测试并显示结果。工程师偶尔通过管理页面配置产品参数。

**核心设计原则：**
- 操作员距离屏幕可能有 50-80cm，文字要大
- PASS/FAIL 结果要在 1 秒内看清楚
- 工厂环境光线复杂，对比度要高
- 操作流程要简单，不能让操作员思考

## 后端 API

后端运行在 `http://localhost:8000`，通过 Vite proxy 转发（配置 `/api → http://localhost:8000`）。

### 接口列表

#### 产品管理

```
GET    /api/products              返回产品列表
GET    /api/products/{code}       返回产品详情
POST   /api/products              新建产品
PUT    /api/products/{code}       更新产品
DELETE /api/products/{code}       删除产品
```

#### 仪器控制

```
POST /api/initialize    初始化仪器（连接+加载配置+验证）
GET  /api/status        当前系统状态
POST /api/test/run      执行一次测试
POST /api/disconnect    断开连接
```

### 关键数据类型

```typescript
// 产品配置
interface Product {
  product_code: string          // 如 "ZZ-T250005A"
  product_name: string          // 如 "乐马六端口辅助源变压器"
  instrument_config_id: number  // 仪器内第几号配置文件
  description: string
  test_items: TestItem[]
  test_items_count?: number     // 列表接口有，详情接口没有
}

interface TestItem {
  test_type: string    // "Turn" | "Lx" | "Q" | "Lk" | "Cx" | "Dcr"
  pins: string         // 如 "3-1"
  description: string
  lower_limit: number | null   // null 表示不限
  upper_limit: number | null
  standard_value: number | null
  unit: string | null  // "mH" | "μH" | "pF" | "Ω" 等
}

// 初始化结果
interface InitStatus {
  ok: boolean
  port?: string          // 如 "COM4"
  idn?: string           // 如 "YouCe Electronics,UC2866XB,Ver 1.0.0"
  config_check?: {
    ok: boolean
    missing: string[]    // JSON 要求但仪器没有的项
    extra: string[]      // 仪器多测了的项（无所谓）
    message: string
  }
  message: string        // 人读的状态描述
}

// 系统状态（每3秒轮询）
interface SystemStatus {
  ready: boolean               // 是否已初始化就绪
  product_code: string | null  // 当前加载的产品
  port: string                 // 端口号
}

// 单次测试结果
interface TestResult {
  ok: boolean
  timestamp: string        // "2026-03-07 20:40:28"
  product_code: string
  overall: string          // "PASS" | "FAIL"
  passed: number           // 通过项数
  failed: number           // 失败项数
  items: MeasuredItem[]
  csv_file: string
}

interface MeasuredItem {
  type: string    // "Turn" | "Lx" | "Q" | "Lk" | "Cx" | "Dcr"
  pins: string    // "3-1"
  value: number   // 测量值（SI 单位，见下方换算表）
  lo: number      // 下限（0 表示不限）
  hi: number      // 上限（0 表示不限）
  result: string  // "Pass" | "Fail"
}
```

### 测量值单位换算（前端显示）

| type | 仪器返回 | 显示换算 | 示例 |
|------|---------|---------|------|
| Turn | 无量纲比值 | 直接显示 6位小数 | 0.145270 |
| Lx | H（亨利） | × 1000 → mH | 1.0330 mH |
| Q | 无量纲 | 直接显示 2位小数 | 133.67 |
| Lk | H（亨利） | × 1,000,000 → μH | 23.20 μH |
| Cx | F（法拉） | × 1e12 → pF | 80.34 pF |
| Dcr | Ω | 直接显示 4位小数 | 2.0376 Ω |

**基本信息区**（4个字段）
- 产品料号：新建时可编辑，编辑时禁用（灰色背景）
- 产品名称：文本输入
- 仪器配置编号：数字输入，说明文字"对应仪器内第几号文件（1-100）"
- 描述：文本输入，可选

**测试项配置区**

表头固定，内容可滚动（最大高度 60vh）

列顺序：`# | 类型 | 引脚 | 说明 | 单位 | 下限值 | 上限值 | [删]`

- **类型**：下拉选择 Turn / Lx / Q / Lk / Cx / Dcr（宽度 80px）
- **引脚**：文本输入，placeholder "3-1"（宽度 70px）
- **说明**：文本输入（弹性宽度）
- **单位**：下拉选择，根据类型自动过滤可选项（宽度 70px）
  - Turn/Q：不显示（或显示"无"）
  - Lx/Lk：H, mH, μH, nH
  - Cx：F, μF, nF, pF
  - Dcr：Ω, mΩ, kΩ
- **下限/上限**：数字输入，空为不限（宽度 80px）
- **删除**：`×` 红色图标按钮

添加测试项按钮在表格顶部右侧，点击在列表末尾添加一行空白项

## 交互细节

### 测试页流程

1. 页面打开 → 自动拉产品列表 → 自动拉系统状态
2. 如果 `status.ready === true`，且有 `status.product_code`：
   - 下拉自动选中该产品
   - 状态卡片显示就绪
   - "开始测试"按钮激活
3. 选择产品 → 点"初始化连接"：
   - 按钮变成 loading 状态（约 3-5 秒）
   - 成功：绿色状态卡片，"开始测试"激活
   - 失败：红色错误信息，按钮恢复可点击
4. 点"开始测试"：
   - 按钮变 loading（约 1.7 秒）
   - 成功：右侧结果区刷新，显示新结果
   - 失败：Toast 错误提示，按钮恢复
5. 显示结果后，"开始测试"立即恢复可点击（准备下一个产品）

### 状态轮询

- 每 3 秒调用一次 `GET /api/status`
- 顶部导航栏的连接指示实时更新
- 如果 `status.ready` 变为 false（仪器断连），右侧显示警告

### 错误处理

- 网络错误：Toast 提示"连接后端失败，请确认服务已启动"
- 仪器错误：在状态卡片内显示错误详情
- 配置不匹配：显示具体缺失的测试项

## 注意事项

1. **不要修改 API**，后端接口已完成，直接调用即可
2. **单位换算在前端做**，后端返回的全是 SI 单位
3. **测试结果 items 顺序**就是仪器返回的顺序，不要重新排序
4. **结果中 lo/hi 为 0** 表示不限制，显示时可以显示为 "—"
5. **PASS 判定**：`overall === 'PASS'`，单项 `result === 'Pass'`（注意大小写）
6. **初始化时仪器会实际触发一次测试**，所以操作员第一次初始化时机器会动，属于正常现象，界面上可以说明"初始化中，仪器将执行一次确认测试"
